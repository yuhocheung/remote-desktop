# 安全政策 Security Policy

远程桌面控制涉及 **远程代码执行（RCE）级攻击面**（输入注入 ≈ 代码执行、屏幕捕获涉及隐私）。我们以严谨态度对待安全问题。

## 负责任披露

请勿公开披露漏洞。请通过以下方式私信报告：

- 邮件：security@example.com（请使用我们的 PGP 公钥加密）
- 或在本仓库提交 **私密安全 Issue**（如平台支持）

我们会尽快确认并修复，并适情致谢。

## 威胁模型摘要

- 云端控制平面为 **半可信**：仅掌握连接元数据，拿不到端到端加密的屏幕内容。
- 端到端加密基于 DTLS-SRTP / DTLS，首次配对需 **带外指纹校验**（safety number / 二维码）以防信令 MITM。
- Host 端常驻 **不可伪造横幅**，由独立高权限服务或 OS 安全注意序列绘制。
